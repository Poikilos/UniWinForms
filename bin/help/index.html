A Complete Open-Source Solution for Forms Design!

This document will contain more information in future updates.


Usage:

Currently The Program will load CSharp files, and look for the other file where partial classes are used (i.e. MainForm.cs and MainForm.Designer.cs), so that you can edit the forms.


Purpose:

This program is intended to be a smart solution to forms design where framework-dependent (i.e. System.Design) programs have failed.  It breaks the code into chunks then processes it manually, so their is no need for code regeneration (one that a designer "eats code"); the requisite absence of syntax errors; nor dependence on special formatting (i.e. "#region Windows Form Designer generated code").  These issues can all be avoided by this departure from the Framework's "prepackaged" form design methodology.  The program uses "fuzzy" detection if form control values, and keeps the "fuzz" when it is finished editing the parameters, so you don't lose your code.
